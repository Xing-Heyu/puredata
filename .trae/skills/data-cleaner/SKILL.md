# 数据清洗Skill（字典核心+合规清洗）

## 核心定位
本Skill专注于打造**极简、零难度、合规导向**的数据清洗能力，全程以「字典/字典列表」为唯一数据载体，适配现有系统且可快速落地。

## 解决的核心痛点
1. **数据格式混乱**：统一处理JSON、CSV、文本等多种格式
2. **清洗操作繁琐**：以字典为核心，大幅降低操作复杂度
3. **版权合规风险**：通过加工重构，构建合规防火墙

## 操作流程

### 1. 统一输入
- **接入格式**：支持JSON、CSV、文本等所有主流数据格式
- **转换方式**：通过免费API/轻量工具，将所有非标准格式统一转换为「字典/字典列表」
- **目标**：实现输入格式归一，彻底解决格式混乱问题

### 2. 核心清洗
- **操作核心**：以字典的键（key）和值（value）为唯一操作对象
- **内置规则**：去重、去空格、空值处理、日期标准化
- **难度控制**：全程仅操作字典，清洗难度趋近于0

### 3. 统一输出
- **输出格式**：固定结构的「字典/字典列表」
- **适配性**：直接对接现有系统，无需二次加工
- **优势**：一次适配、终身复用

### 4. 合规导向
- **操作原则**：不复制、不搬运原始数据，仅进行加工、整理、重构
- **保留内容**：仅保留核心字段与结构
- **输出结果**：全新结构化数据，搭建版权风险防火墙
- **适用场景**：大厂及各类项目的合规需求

### 5. 极简落地
- **技术依赖**：不依赖复杂框架和第三方重型库，仅用Python原生字典操作
- **成本控制**：完全开源，零成本
- **扩展性**：落地成本低、试错成本低、扩展性强

## 使用示例

### 输入示例（CSV转换为字典列表）
```python
# 原始CSV数据
name,age,email
Alice, 25, alice@example.com
Bob,30,bob@example.com
Alice, 25, alice@example.com

# 转换为字典列表
[
  {"name": "Alice", "age": "25", "email": "alice@example.com"},
  {"name": "Bob", "age": "30", "email": "bob@example.com"},
  {"name": "Alice", "age": "25", "email": "alice@example.com"}
]
```

### 清洗示例（去重+去空格）
```python
# 清洗后结果
[
  {"name": "Alice", "age": "25", "email": "alice@example.com"},
  {"name": "Bob", "age": "30", "email": "bob@example.com"}
]
```

### 输出示例（固定结构字典列表）
```python
# 最终输出
[
  {"name": "Alice", "age": 25, "email": "alice@example.com", "processed_at": "2024-01-01T12:00:00"},
  {"name": "Bob", "age": 30, "email": "bob@example.com", "processed_at": "2024-01-01T12:00:00"}
]
```
