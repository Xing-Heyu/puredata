> 版本：v2.1.0 | 更新时间：2026-02-26
> 支持功能：文本生成、图片生成、音频生成（多模态）

## 一、快速部署（Docker）

### 1. 准备服务器

| 配置 | 最低要求 | 推荐配置 |
|------|----------|----------|
| CPU | 2核 | 4核 |
| 内存 | 4GB | 8GB |
| 存储 | 50GB | 100GB SSD |
| 系统 | Ubuntu 20.04+ | Ubuntu 22.04 |

### 2. 安装Docker

```bash
# 安装Docker
curl -fsSL https://get.docker.com | bash

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 3. 部署应用

```bash
# 克隆项目
git clone <your-repo-url>
cd platform

# 配置环境变量
cp .env.example .env
nano .env  # 填写API密钥

# 一键部署
chmod +x deploy.sh
./deploy.sh
```

### 4. 访问服务

- 前台: http://your-server-ip:8000
- 后台: http://your-server-ip:8000/admin
- API: http://your-server-ip:8000/docs

## 三、HTTPS配置（生产环境必须）

### HTTPS是什么？

HTTPS不是资质，是**SSL证书配置**。配置后浏览器会显示小锁🔒，数据传输加密。

| 类型 | 说明 | 费用 |
|------|------|------|
| HTTP | 不加密，浏览器提示"不安全" | 免费 |
| HTTPS | 加密传输，浏览器显示小锁🔒 | 证书免费，域名需购买 |

### 获取SSL证书

**方式一：阿里云免费证书（推荐）**
1. 登录阿里云控制台
2. 搜索"SSL证书" → 购买 → 选择"免费证书"
3. 填写域名 → 验证 → 下载证书（选择Nginx格式）

**方式二：Let's Encrypt（免费自动续期）**
```bash
# 安装certbot
sudo apt install certbot

# 申请证书（需要先有域名指向服务器）
sudo certbot certonly --standalone -d your-domain.com
```

### 配置HTTPS

```bash
# 1. 创建SSL目录
mkdir -p ssl

# 2. 上传证书文件
# ssl/cert.pem - 证书文件
# ssl/key.pem - 私钥文件

# 3. 修改 nginx.conf 中的域名
# 将 localhost 改为 your-domain.com

# 4. 启动生产环境（带HTTPS）
docker-compose --profile production up -d
```

### nginx.conf HTTPS配置示例

```nginx
# HTTP跳转HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS配置
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    
    location / {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 八、安全建议

1. **修改默认密码**：首次登录后立即修改
2. **启用HTTPS**：生产环境必须
3. **配置防火墙**：只开放必要端口
4. **定期备份**：每天自动备份（已内置）
5. **更新依赖**：定期更新安全补丁

### 防火墙配置

```bash
# Ubuntu UFW配置
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
```

### 性能优化

```bash
# 增加文件描述符限制
echo "* soft nofile 65535" >> /etc/security/limits.conf
echo "* hard nofile 65535" >> /etc/security/limits.conf

# 优化内核参数
cat >> /etc/sysctl.conf << EOF
net.core.somaxconn = 1024
net.ipv4.tcp_max_syn_backlog = 1024
EOF
sysctl -p
```
